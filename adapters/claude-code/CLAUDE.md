# LoomKit вЂ” Claude Code Project Instructions

LoomKit is a spec-driven design framework with TDD superpowers. This `CLAUDE.md` sets project-level instructions for Claude Code.

## Workflow Commands

Claude Code commands (slash commands in chat):

| Command | Phase | Action |
|---------|-------|--------|
| `/lk:brainstorm` | brainstorm | Socratic dialogue в†’ `proposal.md` |
| `/lk:spec` | spec | Requirements + WHEN/THEN scenarios в†’ `specs/<name>/spec.md` |
| `/lk:design` | design | Architecture + test strategy в†’ `design.md` |
| `/lk:plan` | plan | Bite-sized TDD tasks в†’ `tasks.md` |
| `/lk:apply` | tdd | RED/GREEN/REFACTOR per task в†’ code + `.traceability.yaml` |
| `/lk:verify` | verify | Coverage gate + test run в†’ `.loomkit-verify.json` |
| `/lk:archive` | archive | Verify gate в†’ merge specs в†’ archive |

## Skill Files

Phase instructions are loaded from `~/.claude/skills/loomkit-*/SKILL.md`.
Run `adapters/claude-code/generate-skills.sh` to install them.

Available skills:
- `loomkit-brainstorm` вЂ” Socratic exploration, proposals
- `loomkit-spec` вЂ” WHEN/THEN requirements
- `loomkit-design` вЂ” Technical design with test strategy
- `loomkit-plan` вЂ” Bite-sized TDD tasks with exact code
- `loomkit-tdd` вЂ” RED/GREEN/REFACTOR enforcement
- `loomkit-verify` вЂ” Coverage gate, test suite run
- `loomkit-archive` вЂ” Spec merge, archive gate

## TDD Enforcement Rules

- **Iron Law:** No production code without a failing test first. If code exists without a test в†’ delete the code, write the test first.
- **RED/GREEN/REFACTOR cycle** per requirement.
- **Traceability:** Every test must reference a spec scenario ID (`SC-001`).
- **Coverage gate:** All SHALL/MUST scenarios must have passing tests before archive.
