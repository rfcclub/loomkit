# Proposal: Brainstorm & Plan

## Motivation
From Superpowers: before writing specs, brainstorm explores intent through Socratic dialogue and proposes 2-3 approaches. Before implementing, plans break work into bite-sized TDD steps with exact code. These are the critical "before coding" phases that OpenSpec lacks.

## Scope
- Brainstorm flow: explore context → ask clarifying questions → propose approaches → present design → write proposal
- Plan format: bite-sized tasks (2-5 min each) with exact code in every step, TDD cycle per step
- No placeholders rule (from Superpowers writing-plans)
- Plan self-review (from Superpowers: spec coverage, placeholder scan, type consistency)

## Non-goals
- Subagent-driven development execution (future)
- Git worktree management (separate concern)
- Visual companion (browser mockups)
